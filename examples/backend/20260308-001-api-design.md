# RESTful API design guidelines

## Background

Need consistent API design patterns across all endpoints.

## Conventions

### Versioning

All endpoints prefixed with `/api/v1/`.

### Error format

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Email is required",
    "details": [{ "field": "email", "reason": "required" }]
  }
}
```

### Pagination

Cursor-based pagination with `?cursor=xxx&limit=20`. Response includes `nextCursor`.
